Zeno · Product UX

Information Architecture & Flow Redesign

A restructure of Zeno's navigation, screen hierarchy, flows, and interaction logic — built around one promise: track every recurring charge and warn you before it hits, without ever handing over a bank login.

ScopeStructure · Flows · States
PlatformiOS mobile app
Tab model5 tabs · center action
StatusProposed redesign
Scope guardrail — this document defines information architecture, screen purpose, content blocks, user flows, and interaction states only. Color, typography, branding, and visual style are intentionally excluded and decided in a later phase. The neutral grayscale here is a blueprint, not a design direction.
01

Principles & Key Decisions

Every structural choice below traces back to the product's reason to exist: trust over automation. The eight requested flow changes, and the decisions taken for each.

Experience principles carried into the structure

Trust is the product
The "no bank login · on-device · we never see your data" narrative is repeated at every point of friction — onboarding, connecting an inbox, importing a statement — never buried in settings.
Fast time-to-value
A brand-new user discovers at least one real subscription in the first session, free, before any paywall appears.
"What's about to charge me, what can I kill"
Home leads with upcoming charges and savings opportunities; the full list and cancellation are always one tap away.
Honesty & easy exit
Nothing is claimed as "done" without confirmation. Exporting data, deleting data, and leaving Zeno are deliberately easy to find.

The eight flow changes → decisions

CHANGE 1Free first scan
First discovery is the onboarding destination, not a feature behind a wall. After sign-up the user is routed straight into a discovery method and sees real results before any paywall. The privacy promise is the backdrop of this moment. Free
CHANGE 2Subs get a home
New dedicated Subscriptions tab holds the complete, searchable, filterable list. Was: only at the bottom of Analytics. Home becomes a triage surface ("what's coming / what to cut"); the full list is one tap away. New tab
CHANGE 3De-clutter Home
The 9-item feature grid is removed. Genuinely useful tools (Analytics, Coach, Spend Twin, Wrapped, Family, Widgets) live inside the Insights tab and Settings. Dev/diagnostic screens move to a hidden area; unbuilt placeholders are pulled from the app. Grid Dev hidden
CHANGE 4Verified cancel
Cancellation gains a verification lifecycle: Guided → Pending verification → re-checks next receipt/statement → Verified cancelled or Still being charged — needs attention. Self-reported "done" is never the end state. Reworked
CHANGE 5Editable date
Manual add gets a user-editable next-renewal date (and optional last-charged date to seed history). Was: locked to +30 days. Fixed
CHANGE 6Real history
Subscription detail shows actual tracked charges with states for none / a few months / long history. Was: mock sample data. Fixed
CHANGE 7Onboarding
Re-sequenced so value comes before configuration. The mode/theme choice is removed from the critical path and offered later as an optional, skippable prompt. Privacy/trust is a multi-beat narrative, not one bullet. Resequenced
CHANGE 8Visible trust
Privacy reassurance is surfaced inline at inbox-connect and CSV-import. Export data, delete all data, and cancel Zeno are top-level in Settings → Data & Privacy, never hidden. Reinforced

Free

04

Primary User Flows

Step-by-step journeys for the six critical paths. ● marks where an upgrade gate can appear — always after value.

A

Brand-new user → first subscription tracked

The make-or-break first session. Value before configuration, value before paywall.

  1. Launch → Welcome (OB-1)
    2–3 trust beats: no bank login, on-device, warned before charges.
  2. Sign up with age gate (OB-2/3)
    Tick 16+, choose magic-link / Apple / Google; verify if magic-link.
  3. First-discovery launchpad (OB-4)
    Pick a method. Privacy reassurance shown inline. No mode choice, no paywall.
    FreeFirst scan is always free.
  4. Run discovery → review results (DSC-R)
    See real subscriptions with confidence indicators; select & edit.
    If user skipped Land on Home empty state that drives back to Discover — never a dead end, never a wall.
  5. Add selected → Home populated
    The "aha": their real subscriptions, next renewals, and this-month total are now visible.
  6. Optional nudges
    Dismissible prompts to enable notifications and (optionally) pick an app mode. Skippable.
B

Discover & add — email-receipt scan

Lowest-effort discovery; trust is the gating concern, so it's addressed inline.

  1. Discover hub → Email scan (DSC-0 → DSC-E)
  2. Trust panel + connect inbox
    Explain read-only, on-device, last-12-months, revocable any time — before the OAuth prompt.
  3. Grant read-only access
    Provider consent screen; option to add more inboxes.
  4. Scanning (loading state)
    On-device progress; cancellable; partial results stream in.
    Error No access / no receipts found → explain & offer CSV or manual instead.
  5. Review & select (DSC-R)
    Confidence-ranked list; edit amount/cycle/date; de-dupe; confirm selection.
  6. Added → Subscriptions + Home update
    ●Repeat/auto re-scan offered here as Pro; first results already delivered free.
C

Discover & add — CSV import

The "most complete" method, for users who'd rather export a statement than connect an inbox.

  1. Discover hub → CSV import (DSC-0 → DSC-C)
  2. Export helper
    Short per-bank "how to export your statement CSV" guidance.
  3. Pick file
    System file picker; trust note that the file is parsed on-device, not uploaded raw.
  4. Parse & detect recurring (loading)
    Identify recurring charges & cadence.
    Error Unreadable / wrong format → show expected columns, link helper, offer manual add.
  5. Review & select (DSC-R)
    Detected recurring charges with confidence; edit & de-dupe; confirm.
  6. Added → list + Home update
D

Cancel a subscription — with verification

The core trust feature. The app never claims a cancellation it hasn't confirmed.

  1. Subscription detail → Cancel (SUB-D → CXL)
  2. Difficulty + guided steps
    Show the service's difficulty rating and step-by-step instructions.
  3. Open cancellation page
    Deep link out to the service's real cancel page; user performs the cancellation.
  4. "Did you cancel it?"
    Self-report. On Yes, the sub moves to Pending verification — not "cancelled."
    State · pending Sub shows Pending verification in the list; Zeno will re-check around the next renewal.
  5. Verification engine re-checks
    At the next receipt scan / statement / renewal date, look for a charge from that service.
    Resolves → verified No charge found → Verified cancelled; show annual savings; move to Cancelled filter.
    Resolves → still charged Charge still present → Still being charged — needs attention; re-alert, re-offer guidance / escalation tips.
E

Upgrade to Pro

Prompted only after the user has felt value — never before the first discovery.

  1. Hit a value ceiling
    Cross the 8-sub Free limit, open a Pro tool (Coach / Wrapped / Spend Twin), or request a repeat auto-scan.
    ●Contextual upgrade entry, tied to the thing the user just tried to do.
  2. Paywall (PAY)
    Monthly / annual Pro with 7-day trial; value list framed around what they just hit; Family option; restore purchases.
  3. Purchase / start trial
    Native purchase sheet; confirm.
    Error / cancel Purchase fails or is dismissed → return to the gated action with Free experience intact; offer restore.
  4. Unlock & return
    Land back exactly where the gate was, now unlocked (e.g. the Coach screen renders).
F

Create or join a household

Privacy-preserving sharing — combined totals only, never line items.

  1. Settings → Family (SET → FAM)
  2. Choose create or join
    Create generates a share code; Join asks for one.
  3. Privacy confirmation
    Explicit "members see your monthly total only — never what you subscribe to" before joining.
  4. Combined view
    Household total + per-member totals (up to 5). Manage / leave household.
    Gate Household is a Family capability — joining/creating routes through the plan if not entitled.

Budgeting Add-on

A subscription-first, forecast-led budgeting layer that never needs a bank login. It's fully useful from tracked subscriptions alone, gets richer with a CSV import, and supports manual envelopes for privacy purists. This extends the app above — it does not redesign it.

Data honesty — budgeting draws on three sources and labels which is which: tracked subscriptions (always available → precise forward forecast), imported CSV ("as of [date]", never live), and manual entries (envelopes you fund + log). No source implies real-time bank tracking.

Where budgeting lives

No new tab — the bar is already five items. Budget gets a dedicated screen reached two ways, plus a glanceable status everywhere spend already appears:

Home status card
New
A glanceable "on pace / approaching / over" card under the spend summary — projected vs cap + mini bar. Taps through to Budget.
Insights → Budget
Primary entry
Budget is the top block of the Insights tab (Insights already owns spend trend + categories), opening the full Budget screen.

Rationale: Home answers "what's about to charge me," Insights owns analysis — budgeting is analysis with a forward target, so it belongs to Insights with a status echo on Home. A sixth tab would crowd the bar and bury the cancellation wedge.

Flows

a

First budget with zero extra data

  1. Home status card / Insights → Budget → "Set a budget"
  2. See live forecast from renewals
    Committed-so-far + still-to-renew, before setting anything. No import.
  3. Pick a cap (or accept suggested)
    Instant headroom feedback against the forecast.
  4. Start tracking → overview with forward status
    FreeOne overall budget + forecast + early alert is free.
b·c

Per-category budgets · manual envelopes

  1. Budget → Category budgets
    Set caps on existing subscription categories; committed spend auto-fills.
    ●Pro
  2. Budget → Manual envelopes → Add
    Name + fund an envelope; "Log" a spend against it. No import needed.
d

Approaching/over → early alert → get back under → cancel

  1. Forecast crosses threshold
    Early alert fires via the existing notification system + quiet hours — ideally before the charge that tips over.
  2. Tap alert → Budget overview (over state)
    "Cut $X to get back under" with cheapest/least-used subs + savings.
  3. Tap Cancel on a sub → existing cancel + verification flow
    Reuses Guided → Pending → Verified, banking the savings back into the budget.
e·f·g

Enrich via CSV · income context · monthly recap

  1. Category budget → Refresh import
    Routes into the existing CSV import; category bars gain a non-subscription portion shown "as of [date]."
  2. Income context → enter monthly income
    Shows subscriptions as % of income + what's left. Entirely optional.
  3. Month rolls over → recap
    Under/over verdict, streak, trend vs cap; feeds Year in Review.
